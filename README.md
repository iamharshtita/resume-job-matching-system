# Resume-Job Matching System

A multi-agent pipeline for resume and job description matching using semantic skill mining, IDF-weighted scoring, and explainable candidate ranking.

---

## What this system does

Traditional resume screening relies on keyword matching, which misses qualified candidates and cannot understand semantic skill equivalences. This system uses a chain of specialized agents to parse resumes and job descriptions, mine skills semantically, score candidates across multiple dimensions, and produce ranked results with plain-English explanations.

---

## Architecture

```
Raw Resume Text              Raw JD Text
       |                          |
       v                          v
 ResumeParserAgent          JDParserAgent
 (skills, experience,       (required/preferred skills,
  english level)             experience years, english level)
       |                          |
       v                          v
 SkillMiningAgent           SkillMiningAgent
 (maps raw skills to        (maps raw skills to
  canonical names via        canonical names via
  FAISS + embeddings)        FAISS + embeddings)
       |                          |
       +------------+-------------+
                    |
                    v
             MatchingAgent
       skill score (0.40)
       experience score (0.30)
       english level score (0.20)
       title overlap score (0.10)
                    |
                    v
        RankingExplanationAgent
        ranked list + explanation per candidate
```

Score weights are configurable in `src/config.py`.

## Installation

Clone the repository and run the setup script. It creates the virtual environment, installs all dependencies, and downloads the raw dataset:

```bash
git clone <repo-url>
cd resume-job-matching-system
python3 setup_env.py
```

This takes 10-15 minutes depending on your internet connection. If you already have the raw data in `data/raw/`, skip the download:

```bash
python3 setup_env.py --skip-download
```

Once setup finishes, activate the virtual environment:

```bash
source venv/bin/activate
```

All commands below assume the virtual environment is active.

---

## Running the pipeline

Run these two commands once. After that the pipeline is ready to use.

### Step 1 — Prepare the dataset

Parses raw data, rebuilds taxonomy, computes IDF weights, and optionally fine-tunes the embedding model. All in one command:

```bash
python3 scripts/run_preprocessing.py
```

To skip fine-tuning (faster, ~10 mins instead of ~40 mins):

```bash
python3 scripts/run_preprocessing.py --skip-finetune
```

Output saved to `data/processed/`, `data/taxonomy/`, `data/test/`. Summary written to `outputs/results/preprocessing_summary.txt`.

### Step 2 — Verify the pipeline

Run a quick end-to-end test using built-in sample data:

```bash
python3 scripts/run_full_pipeline.py
```

## Matching resumes against a job description

Match a single resume against a JD using your own text files:

```bash
python3 scripts/run_full_pipeline.py --resume resume.txt --jd job.txt
```

Rank multiple candidates against one JD:

```bash
python3 scripts/run_full_pipeline.py --rank alice.txt bob.txt carol.txt --jd job.txt
```

Input files must be plain `.txt`. Filename does not matter.

## Using the orchestrator in code

```python
import sys
sys.path.insert(0, "src")
from agents.orchestrator import SkillMiningOrchestrator

orchestrator = SkillMiningOrchestrator()

# score a single pair
result = orchestrator.run(
    resume_text="Skills: Python, FastAPI, PostgreSQL, Docker...",
    jd_text="Requirements:\n- Python\n- FastAPI\n- PostgreSQL",
    resume_id="candidate-001",
    jd_id="job-001",
)

print(f"Final score: {result['final_score']}")

# rank multiple candidates
candidates = [
    ("candidate-001", "Alice ... Skills: Python, Django ..."),
    ("candidate-002", "Bob   ... Skills: JavaScript, React ..."),
]

ranked = orchestrator.rank_candidates(
    jd_text="Senior Python Engineer...",
    candidates=candidates,
    jd_id="job-001",
)

for r in ranked:
    print(f"Rank {r['rank']} | Score {r['final_score']:.3f} | {r['resume_id']}")
    print(f"Matched: {r['matched_skills']}")
    print(f"Missing: {r['missing_skills']}")
    print(r['explanation'])
```

## Troubleshooting

If you see `ModuleNotFoundError`, make sure the virtual environment is active:

```bash
source venv/bin/activate
```

If FAISS fails on Mac, install the CPU version explicitly:

```bash
pip install faiss-cpu
```

If fine-tuning runs out of memory, reduce batch size:

```bash
python3 scripts/run_preprocessing.py --finetune-samples 2000 --finetune-epochs 2 --batch-size 8
```

## Dataset

Djinni dataset (Ukrainian tech job market, sourced from HuggingFace):
- 210,250 resumes filtered to ~81K after quality filtering
- 141,897 job descriptions filtered to ~95K
- 21 technical keyword categories: .NET, C++, DevOps, Flutter, Golang, Java, JavaScript, Node.js, PHP, Python, Ruby, Scala, SQL, iOS, Unity, Data Analyst, Data Engineer, Data Science, Business Analyst, QA, QA Automation
